## 🟢 الطريقة الأولى (المُوصى بها): VHS

### الخطوة 1: ثبّت VHS على جهازك

VHS تحتاج أيضًا برنامجَي `ffmpeg` و `ttyd` (تُثبَّت معها غالبًا).

**على ماك (الأسهل):**
```bash
brew install vhs
```

**على لينكس:**
```bash
# نزّل أحدث إصدار من صفحة الإصدارات، أو عبر Go:
go install github.com/charmbracelet/vhs@latest
# وثبّت المتطلبات:
sudo apt install ffmpeg        # ل ffmpeg
# و ttyd من: https://github.com/tsl0922/ttyd/releases
```

**على ويندوز:**
```powershell
winget install charmbracelet.vhs
# أو استخدم WSL (نظام لينكس داخل ويندوز) واتبع تعليمات لينكس.
```

> **لماذا VHS؟** لأنها تحوّل ملفًا نصّيًا بسيطًا (`.tape`) إلى GIF عالي الجودة،
> بثبات تامّ وبلا أخطاء بشرية في التسجيل. والملف النصّي يبقى في مشروعك، فيمكنك
> تعديل العرض وإعادة توليده في أي وقت.

### الخطوة 2: فعّل البيئة الافتراضية

```bash
source .venv/bin/activate     # ويندوز: .venv\Scripts\Activate.ps1
```
**لماذا؟** حتى يجد VHS الأمر `iocforge` عندما يكتبه السكربت في الطرفية.

### الخطوة 3: شغّل السكربت (أمر واحد!)

```bash
vhs docs/demo.tape
```

سيفتح VHS طرفية وهمية، ويكتب الأوامر بنفسه، ويسجّل كل شيء، ثم يحفظ النتيجة في:
```
docs/img/demo.gif
```

### الخطوة 4: ضع الـ GIF في الـ README

أضف هذا السطر في أعلى ملف `README.md` (تحت العنوان مباشرةً):
```markdown
![IOCForge demo](docs/img/demo.gif)
```
ثم احفظ وارفع التغيير:
```bash
git add docs/img/demo.gif README.md
git commit -m "Add demo GIF"
git push
```

## 🧩 شرح محتوى السكربت `demo.tape`

السكربت مقسّم إلى "مشاهد" (Scenes). إليك ماذا يحدث في كل سطر:

| التعليمة | ماذا تفعل؟ |
|---|---|
| `Output docs/img/demo.gif` | مكان حفظ الناتج |
| `Set Theme "Dracula"` | ثيم ألوان داكن أنيق |
| `Set FontSize 18` | حجم الخط |
| `Set Width / Height` | أبعاد الفيديو بالبكسل |
| `Set TypingSpeed 55ms` | سرعة "كتابة" الأوامر (لتبدو واقعية) |
| `Type "..."` | يكتب نصًّا في الطرفية |
| `Enter` | يضغط Enter (ينفّذ الأمر) |
| `Sleep 3s` | ينتظر 3 ثوانٍ (ليقرأ المشاهد المخرجات) |

### سيناريو العرض (الرحلة الكاملة):
1. **عنوان تعريفي** للأداة.
2. **`iocforge info`** ← يُظهر الشعار ASCII والقدرات والخدمات المُحمّلة.
3. **`iocforge analyze ...`** ← يحلّل تقريرًا تجريبيًا ويُظهر الملخّص (21 مؤشرًا).
4. **`ls reports/`** ← يُظهر التقارير الأربعة المُولّدة.
5. **`open ...html`** ← فتح لوحة HTML التفاعلية في المتصفّح.
6. **سطر ختامي** يلخّص الإنجاز.

> 🎛️ **تريد تعديل العرض؟** افتح `docs/demo.tape` وغيّر النصوص أو أزمنة
> `Sleep` أو الثيم، ثم أعد تشغيل `vhs docs/demo.tape`. هذه قوة VHS.

## 🟡 الطريقة الثانية (بديل): asciinema

إن لم ترغب في تثبيت VHS، يمكنك تسجيل جلسة حقيقية:

```bash
# 1) ثبّت asciinema
brew install asciinema           # أو: pip install asciinema

# 2) سجّل جلسة (نفّذ الأوامر يدويًا أثناء التسجيل)
asciinema rec demo.cast

#    اكتب بداخلها:
#      iocforge info
#      iocforge analyze examples/sample_report.txt --no-enrich -q
#      ls reports/
#    ثم اكتب exit لإنهاء التسجيل.

# 3) حوّل التسجيل إلى GIF عبر أداة agg
#    (ثبّتها من: https://github.com/asciinema/agg)
agg demo.cast docs/img/demo.gif
```

**الفرق:** asciinema تسجّل **جلستك الحقيقية** (مرن لكن قد يحتاج محاولات)، بينما
VHS تكتب الأوامر تلقائيًا من السكربت (ثابت ونظيف). **نوصي بـ VHS.**
