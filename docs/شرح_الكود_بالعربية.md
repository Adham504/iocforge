## 1) ما هو المشروع باختصار؟

**IOCForge** أداة تأخذ ملفات (تقارير، سجلّات، PDF…) وتستخرج منها
"مؤشرات الاختراق" (Indicators of Compromise – IoCs) مثل عناوين IP،
النطاقات (Domains)، الروابط، البريد الإلكتروني، بصمات الملفات (Hashes)…

ثم:
- تنظّف النتائج وتحذف **الإيجابيات الكاذبة** (مثل `127.0.0.1` أو `example.com`).
- تستعلم عن كل مؤشر من خدمات استخبارات تهديدات (VirusTotal، AbuseIPDB…).
- تصدّر تقارير بصيغ JSON و CSV و HTML وملخّص نصّي.

```
ملف ملوّث  ──▶  IOCForge  ──▶  قائمة مؤشرات نظيفة + تقييم خطورة + تقارير جميلة
```

## 2) الصورة الكبيرة: كيف تسير البيانات؟

```
 ┌────────┐   ┌──────────────┐   ┌───────────────────┐   ┌──────────────────┐   ┌──────────────┐
 │ ملفات  │──▶│ ParserFactory│──▶│ ExtractorRegistry │──▶│ EnrichmentManager│──▶│ ReportManager│
 └────────┘   └──────────────┘   └───────────────────┘   └──────────────────┘   └──────────────┘
   أي صيغة       نصّ عادي            كائنات Indicator         تقييم الخطورة          JSON/CSV/HTML
                                    (بلا تكرار + عدّاد)        (طلبات متوازية)        + ملخّص
```

اقرأ المخطط من اليسار لليمين:
1. **ParserFactory**: يحوّل أي ملف إلى **نصّ عادي**.
2. **ExtractorRegistry**: يبحث في النصّ عن المؤشرات بتعابير نمطية (Regex).
3. **EnrichmentManager**: يسأل خدمات الإنترنت عن خطورة كل مؤشر.
4. **ReportManager**: يكتب التقارير النهائية.

والذي يربط كل هذا معًا هو **المحرّك** `IOCForgeEngine`.

## 4) طبقة الإعدادات (config)

### الملف: `config/settings.py`

**لماذا يوجد؟** لكي نضع كل الإعدادات (مفاتيح الـ API، المهلة الزمنية…) في
مكان واحد، وبشكل **آمن** (المفاتيح تُقرأ من ملف `.env` ولا تُكتب داخل الكود).

**كيف يتواصل مع الباقي؟** كل وحدة تحتاج إعدادًا تستدعي `get_settings()`.

#### شرح الكود

```python
@dataclass(frozen=True)
class Settings:
    virustotal_api_key: Optional[str] = None
    ...
```
- `@dataclass` تُولّد لنا تلقائيًا دوال البناء (`__init__`) والمقارنة.
- `frozen=True` تعني أن الكائن **غير قابل للتعديل** بعد إنشائه (للأمان).

```python
@property
def loaded_apis(self) -> list[str]:
    apis = []
    if self.virustotal_api_key:
        apis.append("VirusTotal")
    ...
    return apis
```
- `@property` تجعل `loaded_apis` يُستخدم كأنه متغيّر لا دالة.
- الفكرة: نرجّع أسماء الخدمات التي **يوجد لها مفتاح فعليًا** فقط.

```python
@lru_cache(maxsize=1)
def get_settings() -> Settings:
    return Settings(
        virustotal_api_key=os.getenv("VIRUSTOTAL_API_KEY") or None,
        ...
    )
```
- `os.getenv("X")` يقرأ متغيّر البيئة `X` (يأتي من ملف `.env`).
- `@lru_cache(maxsize=1)` تعني: احسب الإعدادات **مرة واحدة** ثم احفظ النتيجة
  (لتفادي قراءة البيئة مرارًا).

> 🔐 **درس أمني:** لا تكتب مفاتيح الـ API داخل الكود أبدًا. ضعها في `.env`
> الموجود في `.gitignore` حتى لا تُرفع إلى GitHub.

## 6) النماذج الأساسية (core/models.py)

**لماذا يوجد؟** هذه الملفات تعرّف "شكل البيانات" التي تتنقّل بين كل أجزاء
المشروع. عندما يتفق الجميع على نفس الشكل، يصبح التواصل سهلًا.

### `IOCType` (نوع المؤشر)

```python
class IOCType(str, enum.Enum):
    IPV4 = "ipv4"
    DOMAIN = "domain"
    ...
```
- `Enum` = قائمة ثابتة من القيم المسموح بها. بدل أن نكتب النصّ `"ipv4"` في
  كل مكان (وقد نخطئ كتابته)، نستخدم `IOCType.IPV4`.

### `RiskLevel` (مستوى الخطورة)

```python
class RiskLevel(str, enum.Enum):
    CLEAN = "clean"
    ...
    @property
    def color(self):   # لون لكل مستوى يُستخدم في تقرير HTML
        return {RiskLevel.CRITICAL: "#8e44ad", ...}[self]
    @property
    def rank(self):    # رقم للترتيب: كلما زاد كان أخطر
        return {RiskLevel.CRITICAL: 5, ...}[self]
```

### `Indicator` (المؤشر الواحد)

```python
@dataclass
class Indicator:
    value: str            # القيمة، مثل "8.8.8.8"
    ioc_type: IOCType     # نوعها
    sources: List[str]    # في أي ملفات وُجدت
    count: int = 1        # كم مرة ظهرت
    enrichment: List[EnrichmentResult]  # نتائج الاستعلام من الخدمات
```

```python
@property
def risk_level(self) -> RiskLevel:
    # 1) تجاهل المزوّدين الذين فشلوا أو أرجعوا UNKNOWN
    verdicts = [e.risk_level for e in self.enrichment
                if e.error is None and e.risk_level is not RiskLevel.UNKNOWN]
    if not verdicts:
        return RiskLevel.UNKNOWN
    worst = max(verdicts, key=lambda r: r.rank)
    # 2) إذا قال مزوّد "موثوق" (مثل VirusTotal) إنه نظيف، وكان أسوأ تقييم
    #    لا يتجاوز LOW، نعتبره نظيفًا (نكبح الإشارة الضعيفة المنفردة)
    authoritative_clean = any(
        e.risk_level is RiskLevel.CLEAN
        and e.provider in self._AUTHORITATIVE_PROVIDERS
        for e in self.enrichment)
    if authoritative_clean and worst.rank <= RiskLevel.LOW.rank:
        return RiskLevel.CLEAN
    return worst
```

**لماذا هذا المنطق الذكي؟ (إصلاح مهم)**
- المنطق القديم كان: "خطورة المؤشر = **أسوأ** تقييم أعطته أي خدمة". لكن هذا
  يسبّب **إنذارات كاذبة**! مثال حقيقي: بصمة الملف الفارغ (`e3b0c442...`) تظهر
  في مئات نبضات OTX (كأثر جانبي بريء)، فيعطيها OTX تقييمًا مرتفعًا، بينما
  VirusTotal يقول بوضوح "0 من 75 محرّكًا اعتبره خبيثًا" أي **نظيف**.
- الحل: إذا قال مزوّد **موثوق متعدّد المحرّكات** (VirusTotal أو AbuseIPDB) إنه
  **نظيف**، فإن إشارة ضعيفة منفردة (LOW) من مزوّد تخميني (مثل عدّاد نبضات OTX)
  **لا تَرفع** مستوى الخطورة. أمّا التهديدات الحقيقية (MEDIUM فأعلى) فتظهر دائمًا.
- بالإضافة لذلك: بصمات الملف الفارغ تُحذف أصلًا في `false_positives.py`
  (دالة `is_benign_hash`) قبل أن تصل للإثراء.

> 🎨 **ملاحظة عن الألوان:** مخرجات الطرفية الآن **ملوّنة** حسب الخطورة (أخضر
> للنظيف، أحمر للخطير...) عبر `utils/colors.py`، مع احترام `NO_COLOR` وعدم
> التلوين عند توجيه المخرجات إلى ملف.

## 7) المُستخرِجات (extractors)

هذا قلب الأداة: **كيف نتعرّف على المؤشرات داخل النصّ؟**

### `extractors/patterns.py` — التعابير النمطية (Regex)

**لماذا يوجد؟** ليجمع كل أنماط الـ Regex في مكان واحد + دالة "فكّ التمويه".

#### فكّ التمويه (refang)

المهاجمون يكتبون الروابط "ملغومة" مثل `hxxp://1[.]2[.]3[.]4` حتى لا تضغط
عليها بالخطأ. قبل البحث يجب أن نعيدها لشكلها الطبيعي:

```python
_REFANG_REPLACEMENTS = [
    (re.compile(r"h[xX]{2}p(s?)://", re.IGNORECASE), r"http\1://"),
    (re.compile(r"\[\.\]"), "."),
    (re.compile(r"\[at\]", re.IGNORECASE), "@"),
    ...
]
def refang(text):
    for pattern, replacement in _REFANG_REPLACEMENTS:
        text = pattern.sub(replacement, text)
    return text
```
- لكل نمط ملغوم نمط بديل. `sub` تستبدل كل التطابقات.
- مثال: `hxxp://bad[.]net` ➜ `http://bad.net`.

#### أمثلة على الأنماط

```python
MD5    = re.compile(r"\b[a-fA-F0-9]{32}\b")   # 32 رمزًا ست عشريًا
SHA1   = re.compile(r"\b[a-fA-F0-9]{40}\b")   # 40
SHA256 = re.compile(r"\b[a-fA-F0-9]{64}\b")   # 64
CVE    = re.compile(r"\bCVE-\d{4}-\d{4,7}\b", re.IGNORECASE)
```
- `\b` = حدّ كلمة (يمنع التطابق في منتصف كلمة).
- `{32}` = بالضبط 32 رمزًا.

> 💡 الـ IPv6 صعب لأن له اختصار `::`. لذلك نمطه طويل، ثم نتحقق من صحته لاحقًا
> بمكتبة `ipaddress`.

### `extractors/base.py` — القالب الأساسي

**لماذا يوجد؟** ليفرض على كل مُستخرِج نفس "الواجهة" (نفس الدوال). أي مستخرِج
جديد يرث من `BaseExtractor`.

```python
class BaseExtractor(abc.ABC):
    ioc_type: IOCType

    @abc.abstractmethod
    def extract(self, text) -> List[str]: ...   # يبحث عن المرشّحين

    def is_valid(self, candidate) -> bool:       # هل هو حقيقي أم كاذب؟
        return True

    def normalize(self, candidate) -> str:       # توحيد الشكل (حروف صغيرة…)
        return candidate.strip()

    def find(self, text) -> Set[str]:            # يجمع الثلاثة معًا
        results = set()
        for c in self.extract(text):
            n = self.normalize(c)
            if n and self.is_valid(n):
                results.add(n)
        return results
```
- `abc.ABC` + `@abc.abstractmethod` تعني: لا يمكن إنشاء `BaseExtractor`
  مباشرة؛ يجب أن ترث منه وتكتب `extract`.
- `find` هي القصة كاملة: استخرج ➜ وحّد ➜ تحقّق ➜ احذف التكرار (بـ `set`).

### `extractors/false_positives.py` — حذف الكاذب

**لماذا يوجد؟** كثير من المؤشرات ليست خطيرة (IP داخلي، نطاق وهمي). نحذفها هنا.

```python
def is_public_ip(value) -> bool:
    try:
        ip = ipaddress.ip_address(value)
    except ValueError:
        return False
    if (ip.is_private or ip.is_loopback or ip.is_reserved
            or ip.is_multicast or ip.is_link_local or ip.is_unspecified):
        return False
    return ip.is_global
```
- **هنا نستخدم مكتبة `ipaddress` بدل Regex** كما طُلب: فهي تعرف بدقّة ما إذا
  كان العنوان خاصًّا (`192.168.x.x`)، أو محلّيًا (`127.0.0.1`)، أو محجوزًا…
- نقبل العنوان فقط إذا كان `is_global` (قابلًا للتوجيه على الإنترنت).

```python
def is_fake_domain(domain) -> bool:
    domain = domain.lower().rstrip(".")
    if domain in FAKE_DOMAINS:        # مثل example.com
        return True
    tld = domain.rsplit(".", 1)[-1]
    if tld in NON_DOMAIN_TLDS:        # مثل .py .exe .pdf (أسماء ملفات!)
        return True
    return False
```
- `FAKE_DOMAINS`: قائمة نطاقات شهيرة للأمثلة (نتجاهلها).
- `NON_DOMAIN_TLDS`: امتدادات ملفات قد تبدو كنطاقات (مثل `script.py`).

### `extractors/registry.py` — المنظّم

**لماذا يوجد؟** ليشغّل **كل** المستخرِجات على النصّ بترتيب ذكي.

**المشكلة التي يحلّها:** بصمة SHA256 طولها 64 رمزًا، وبداخلها يوجد 40 رمزًا
(تبدو SHA1) و32 رمزًا (تبدو MD5)! لو لم ننتبه سنعدّ البصمة الواحدة 3 مرات.

```python
def extract_all(self, text):
    text = patterns.refang(text)      # 1) أعِد الروابط الملغومة
    results = {}
    masked = text
    # 2) استخرج البصمات من الأطول للأقصر، ثم "اخفِها" من النصّ
    for ioc_type in (IOCType.SHA256, IOCType.SHA1, IOCType.MD5):
        found = extractor.find(masked)
        for value in found:
            masked = re.sub(re.escape(value), " ", masked)  # إخفاء
    # 3) ثم بقية الأنواع
    ...
```
- نخفي البصمات الطويلة أولًا حتى لا تُعَدّ ثانيةً كبصمات أقصر. هذا حلّ أنيق.

```python
def get_default_registry():
    return ExtractorRegistry([IPv4Extractor(), IPv6Extractor(), ...])
```
- دالة جاهزة تبني المنظّم بكل المستخرِجات.

## 8) قارئات الملفات (parsers)

**لماذا يوجد؟** كل صيغة ملف تُقرأ بطريقة مختلفة. هدفنا تحويل أي ملف إلى **نصّ**.

### `parsers/base.py`

```python
@dataclass
class ParsedDocument:
    source: str                     # اسم الملف
    text: str                       # محتواه نصًّا
    children: List["ParsedDocument"]# للملفات داخل ZIP
    def flatten(self):              # يرجّع الملف + كل أبنائه في قائمة واحدة
        docs = [self]
        for child in self.children:
            docs.extend(child.flatten())
        return docs
```

```python
class BaseParser(abc.ABC):
    extensions: tuple = ()
    @abc.abstractmethod
    def parse(self, path) -> ParsedDocument: ...
```

### `parsers/text_parsers.py`

- `PlainTextParser`: يقرأ `.txt`/`.log` كنصّ مباشر.
- `CSVParser`: يقرأ الجداول ويحوّل كل صفّ إلى سطر نصّي.
- `JSONParser`: يمشي داخل JSON (قاموس/قائمة) ويجمع كل القيم:

```python
def walk(obj):
    if isinstance(obj, dict):
        for key, value in obj.items():
            chunks.append(str(key)); walk(value)
    elif isinstance(obj, list):
        for item in obj: walk(item)
    else:
        chunks.append(str(obj))
```
هذه **دالة تكرارية (recursive)** تنزل لكل المستويات.

- `HTMLParser`: يزيل وسوم HTML (`<...>`) ويحتفظ بالنصّ + الروابط في
  خصائص مثل `href`. يستخدم مكتبة BeautifulSoup إن وُجدت، وإلا Regex بسيط.

### `parsers/document_parsers.py`

- `PDFParser`: يستخرج النصّ من PDF عبر مكتبة `pypdf`.
- `DOCXParser`: يستخرج النصّ من Word عبر `python-docx`.
- `ZIPParser`: يفكّ ضغط ZIP **ويقرأ كل ملف بداخله** (تكراريًا):

```python
class ZIPParser(BaseParser):
    def __init__(self, factory):    # يحتاج المصنع ليقرأ الملفات الداخلية
        self._factory = factory
    def parse(self, path):
        with zipfile.ZipFile(path) as archive:
            for name in archive.namelist():
                extracted = Path(archive.extract(name, tmp_path))
                if self._factory.supports(extracted):
                    child = self._factory.parse(extracted)
                    root.children.append(child)
```

### `parsers/factory.py` — المصنع

**لماذا يوجد؟** ليختار القارئ الصحيح **تلقائيًا** حسب امتداد الملف.

```python
class ParserFactory:
    def __init__(self):
        self._parsers = [PlainTextParser(), CSVParser(), JSONParser(),
                         HTMLParser(), PDFParser(), DOCXParser()]
        self._parsers.append(ZIPParser(self))   # يمرّر نفسه للـ ZIP
        self._by_ext = {ext: p for p in self._parsers for ext in p.extensions}

    def parse(self, path):
        parser = self.get_parser(path)          # اختر حسب الامتداد
        return parser.parse(path)
```
- `_by_ext` قاموس يربط الامتداد (مثل `.pdf`) بالقارئ المناسب.

## 9) طبقة الإثراء (enrichment)

**لماذا توجد؟** بعد استخراج المؤشرات نريد أن نعرف: هل هي خطيرة فعلًا؟
نسأل خدمات استخبارات التهديدات على الإنترنت.

### `enrichment/base.py`

```python
class BaseEnricher(abc.ABC):
    name = "base"
    supported_types: Set[IOCType] = set()

    def enrich(self, value, ioc_type) -> EnrichmentResult:
        try:
            return self._query(value, ioc_type)   # الاستعلام الفعلي
        except requests.RequestException as exc:
            return EnrichmentResult(provider=self.name, ioc_value=value, error=str(exc))
```
- `enrich` **لا ترمي أخطاء أبدًا**؛ إن فشل الإنترنت تُسجّل الخطأ داخل النتيجة.
  هذا مهم حتى لا يتوقّف البرنامج بسبب مؤشر واحد.

```python
@staticmethod
def _risk_from_detections(malicious, total) -> RiskLevel:
    if total <= 0: return RiskLevel.UNKNOWN
    if malicious == 0: return RiskLevel.CLEAN
    ratio = malicious / total
    if malicious >= 10 or ratio >= 0.5: return RiskLevel.CRITICAL
    ...
```
- دالة مساعدة تحوّل "نسبة الكشف" (كم محرّك اعتبره خبيثًا) إلى مستوى خطورة.

### مزوّدو الخدمة

- `virustotal.py`: يستعلم من VirusTotal (سمعة، نسبة الكشف، التصنيفات).
- `abuseipdb.py`: أفضل خدمة مجانية لعناوين IP (نسبة ثقة الإساءة، الدولة، المزوّد).
- `otx.py`: AlienVault OTX (عدد النبضات Pulses، عائلات البرمجيات، مجموعات APT).
- `threatfox.py`: ThreatFox + MalwareBazaar من abuse.ch.

كل واحد يكتب فقط دالة `_query` ويحدّد `supported_types` (أنواع المؤشرات
التي يدعمها).

### `enrichment/manager.py` — المدير

**لماذا يوجد؟** ليقرّر **أي مزوّد** يدعم **أي نوع** مؤشر، ويشغّلهم بالتوازي.

```python
def enrich_all(self, indicators):
    if not self.active or not self.settings.enable_enrichment:
        return indicators                       # تخطّى إن لا مفاتيح
    targets = [i for i in indicators if i.ioc_type in ENRICHABLE_TYPES]
    with ThreadPoolExecutor(max_workers=workers) as pool:
        futures = {pool.submit(self.enrich_indicator, ioc): ioc for ioc in targets}
        for future in as_completed(futures):
            future.result()
```
- `ThreadPoolExecutor` يشغّل عدة استعلامات **في نفس الوقت** (أسرع بكثير).
- `as_completed` ينتظر انتهاء كل مهمة.

## 10) المحرّك (core/engine.py)

**لماذا يوجد؟** هو "المايسترو" الذي يربط كل الطبقات السابقة في خطّ إنتاج واحد.

```python
def analyze(self, paths, enrich=None):
    documents = self._gather_documents(paths)   # 1) اقرأ الملفات إلى نصّ
    indicators = self._extract(documents)        # 2) استخرج المؤشرات
    if should_enrich:
        manager.enrich_all(flat)                 # 3) أثرِها (اختياري)
    return AnalysisReport(indicators=indicators, ...)  # 4) ابنِ التقرير
```

دالة الاستخراج تجمع التكرارات وتعدّها:

```python
def _extract(self, documents):
    accumulator = defaultdict(dict)              # نوع ➜ {قيمة: Indicator}
    for doc in documents:
        found = self.registry.extract_all(doc.text)
        for ioc_type, values in found.items():
            for value in values:
                if value in bucket:              # موجود من قبل؟
                    bucket[value].count += occ   # زِد العدّاد
                    bucket[value].sources.append(doc.source)
                else:
                    bucket[value] = Indicator(...)  # جديد
```
- `defaultdict(dict)` يُنشئ قاموسًا فرعيًا تلقائيًا لكل نوع.
- نتأكّد أن نفس المؤشر لا يتكرّر، بل نزيد عدّاده ونضيف مصدره.

> ملاحظة عن **الاستيراد الدائري**: ملف `core/__init__.py` يستورد المحرّك
> "بشكل كسول" (lazy) عبر `__getattr__`، لأن المحرّك يعتمد على المستخرِجات،
> والمستخرِجات تعتمد على `models`. هذا يكسر الحلقة.

## 11) التقارير (reporting)

كل مُصدِّر يرث من `BaseReporter` وينفّذ `render` (يرجّع نصًّا) و`write`
(يكتبه في ملف).

- `json_reporter.py`: يحوّل `report.to_dict()` إلى JSON منسّق.
- `csv_reporter.py`: صفّ لكل مؤشر (للإكسل).
- `summary_reporter.py`: ملخّص نصّي للإنسان (يظهر في الـ Terminal).

```python
def _bar_chart(self, data, title):
    max_val = max(v for _, v in items) or 1
    for idx, (label, value) in enumerate(items):
        bar_w = int((value / max_val) * width)   # طول العمود نسبةً للأكبر

```

## 13) كيف أضيف ميزة جديدة؟

بفضل التصميم (مبدأ Open/Closed)، الإضافة سهلة بدون تعديل الكود القديم:

| أريد أن أضيف… | الخطوات |
|---|---|
| **نوع مؤشر جديد** | اكتب صنفًا يرث `BaseExtractor`، ثم أضِفه في `get_default_registry`. |
| **صيغة ملف جديدة** | اكتب صنفًا يرث `BaseParser`، ثم أضِفه في `ParserFactory`. |
| **خدمة تهديدات جديدة** | اكتب صنفًا يرث `BaseEnricher`، ثم أضِفه في `EnrichmentManager`. |
| **صيغة تقرير جديدة** | اكتب صنفًا يرث `BaseReporter`، ثم أضِفه في `ReportManager`. |

### مثال عملي: إضافة مستخرِج "أرقام هواتف"

```python
# 1) نمط في patterns.py
PHONE = re.compile(r"\+?\d[\d\s\-]{7,}\d")

# 2) صنف جديد
from iocforge.extractors.base import BaseExtractor
from iocforge.core.models import IOCType   # (أضِف PHONE إلى IOCType أولًا)

class PhoneExtractor(BaseExtractor):
    ioc_type = IOCType.PHONE
    def extract(self, text):
        return PHONE.findall(text)

# 3) سجّله
registry.register(PhoneExtractor())
```

انتهى! المحرّك سيشغّله تلقائيًا.
